# Contributing Guide

**AI for Social Good Hackathon – SUST 2026**

This guide is for **organizers, volunteers, and contributors** who help maintain and improve this repository.

---

## Who Should Contribute

- **Organizers:** Update event details, schedule, and operational docs.
- **Volunteers:** Improve documentation clarity, fix links, or add resources.
- **Technical contributors:** Improve starter kits, example projects, or the website.

---

## Contribution Workflow

1. **Fork** this repository to your GitHub account.
2. **Create a branch** with a clear, descriptive name:
   ```
   git checkout -b feature/update-judging-rubric
   ```
3. Make your changes, keeping them focused and minimal.
4. **Test** any website or script changes locally before submitting.
5. **Open a pull request** with a clear title and description.

---

## What to Contribute

| Area | Examples |
|---|---|
| Documentation | Fix typos, clarify rules, update TBD placeholders once dates are confirmed |
| Starter kits | Improve setup instructions, fix bugs in starter code |
| Website | Visual polish, mobile layout improvements, accessibility |
| Resources | Add free tools, APIs, or tutorials useful for participants |
| Outreach materials | Improve poster text, social media templates |

---

## Standards

- Keep all content beginner-friendly and accessible.
- Use professional, clear language throughout.
- Prefer free and open-source tools in all recommendations.
- Do not include real dates unless confirmed by the organizer.
- Keep TBD placeholders where information is not yet finalized.
- Do not add internal planning notes or operational checklists to public-facing files.

---

## Need Help?

Contact the organizer via RoboSUST Club official channels or open a GitHub Issue for documentation questions.
