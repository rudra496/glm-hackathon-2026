# Rules and Participation Policy

## 1) Eligibility

- Participants must be currently enrolled university students (Bangladesh-based primary target).
- Team size: **1 to 4 members**.
- One participant may join only one team.

## 2) Allowed Project Types

- AI-powered web apps, mobile apps, bots, automation tools, education tools, research prototypes, or productivity systems.
- Projects must be functional at prototype level.

## 3) Build Window

- Official development period: **7 days**.
- Prior learning and reusable open-source libraries are allowed.
- Reusing old project code is not allowed beyond starter boilerplate.

## 4) Required Deliverables

Each team must provide:

1. Public GitHub repository
2. Clear `README.md`
3. Demo video (2â€“5 minutes)
4. Submission form response completed before deadline

## 5) GitHub Requirements

Repository must include:

- Project title and problem statement
- Setup/run instructions
- Tech stack
- Team member names and roles
- Commit history from hackathon period
- License file (recommended: MIT)

## 6) Prohibited Actions

Teams may be disqualified for:

- Plagiarism or copying substantial code without attribution
- Submitting non-functional or unrelated repositories
- False claims about authorship or team composition
- Harassment or misconduct in official channels
- Missing deadline

## 7) Use of AI Tools

- AI-assisted development is allowed.
- Teams remain fully responsible for correctness, originality, and licensing.
- Disclose major AI usage in README under a section named `AI Assistance`.

## 8) Judging and Decisions

- Scoring follows the rubric in `JUDGING.md`.
- Organizers may request clarifications.
- Organizer/judge decisions are final.

## 9) Intellectual Property

- Teams retain ownership of their code.
- By participating, teams allow organizers to showcase project summaries, names, and demo links for non-commercial event reporting.
