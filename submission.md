# Submission Guidelines

## What to Submit
Each team must submit in `participants/team-submissions.md`:
1. Team name and member list
2. GitHub repository URL
3. Problem statement (max 150 words)
4. Solution summary (max 200 words)
5. z.ai usage details
6. Demo link (video or live)

## Repository Requirements
- Public repository preferred
- Include a complete `README.md` with:
  - setup steps
  - how to run
  - key features
  - screenshots (optional)

## Demo Guidelines
- Duration: **2–5 minutes**
- Must show:
  - problem context
  - live feature walkthrough
  - z.ai integration highlights
  - expected impact

## Deadline Rules
- Submit before the posted deadline in [schedule.md](./schedule.md)
- One final submission per team
