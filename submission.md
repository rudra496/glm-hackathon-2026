# Submission Guidelines

## 1) Final Submission Requirements
Each team must submit all of the following:
1. Team name + member list
2. GitHub repository URL
3. Problem statement (max 150 words)
4. Solution summary (max 200 words)
5. z.ai usage explanation
6. Demo link (video or live)

Use: `participants/team-submissions.md`

## 2) Repository Quality Requirements
Your project repo should include:
- `README.md` with setup + run steps
- Problem definition and target users
- Architecture or flow diagram (optional but recommended)
- Demo screenshots or GIF (optional)
- License (recommended)

## 3) Demo Guidelines
- Duration: **2–5 minutes**
- Must cover:
  - problem and user pain point
  - live flow / working demo
  - z.ai integration point
  - expected social impact

## 4) Deadline & Integrity
- Late submissions may be penalized
- One final submission per team
- Plagiarism may result in disqualification

## 5) Submission Formatting Tip
Before final submit, verify your entry includes clickable links and complete team identity information.
