# Resources

## Core Stack
- z.ai tools and APIs (official onboarding provided during kickoff)
- GitHub for code hosting and collaboration
- Free tooling only (VS Code, Python, browser-based testing)

## Recommended Learning Path (Beginner-Friendly)
1. Read `README.md` for event context
2. Review [rules.md](./rules.md)
3. Choose a starter kit from `starter-kits/`
4. Build one small end-to-end feature first
5. Polish UX, reliability, and demo flow
6. Submit using [submission.md](./submission.md)

## Build Strategy for 1-Day Hackathon
- **Hour 1–2:** Problem definition + scope lock
- **Hour 3–4:** Build MVP and integrate z.ai
- **Hour 5:** Add UX polish + edge case handling
- **Hour 6:** Prepare demo and README

## Technical Checklist
- [ ] Clear social problem statement
- [ ] Meaningful z.ai usage
- [ ] Working MVP
- [ ] Clean README with run instructions
- [ ] Demo plan ready (2–5 mins)
