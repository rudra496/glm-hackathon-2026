# Submission Guidelines

**AI for Social Good Hackathon – SUST 2026**

---

## Required Submission Items

Each team must submit **all of the following** before the official deadline:

| # | Item | Details |
|---|---|---|
| 1 | Team information | Team name + full name and department of each member |
| 2 | GitHub repository URL | Must be **public** at time of submission |
| 3 | Problem statement | Max 150 words |
| 4 | Solution summary | Max 200 words |
| 5 | Z.ai usage explanation | What Z.ai tools/APIs were used and how |
| 6 | Demo link | Video (2–5 min) or live URL |

Submit using the **submission form** — link will be announced on event day.

---

## Your Project Repository Must Include

- `README.md` with:
  - Project title and problem statement
  - Solution description and target users
  - Setup and run instructions
  - Tech stack and tools used
  - Team member names and roles
  - Z.ai integration section
- Working source code
- License file (MIT recommended)

---

## Demo Guidelines

- **Duration:** 2–5 minutes
- **Cover:**
  - The problem and who it affects
  - A live walkthrough of your working solution
  - How Z.ai is integrated
  - Expected social impact
- Video (unlisted YouTube or Google Drive) or live demo both accepted.

---

## Deadline & Integrity

- **One final submission per team.** Do not submit multiple times.
- Late submissions may be penalized or rejected at organizer discretion.
- All submitted repositories must be **public** and accessible by judges.
- Plagiarism or misrepresentation will result in disqualification.

---

## After Submitting

Add your team's entry to [`participants/team-submissions.md`](../participants/team-submissions.md) in this repository once your submission is complete.
