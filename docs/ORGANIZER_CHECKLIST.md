# Organizer Checklist

**AI for Social Good Hackathon – SUST 2026**

Use this checklist to prepare for and run the event. Mark items as complete as you go.

---

## Phase 1: Pre-Event Setup (4+ Weeks Before)

### Logistics
- [ ] Confirm event date, time, and venue with SUST
- [ ] Update README.md, docs, and website with confirmed date and venue
- [ ] Book/confirm the venue room and any equipment (projector, microphones, tables)
- [ ] Confirm Z.ai support/contact for API access and introductory session
- [ ] Define registration capacity and set a cutoff

### Registration
- [ ] Create registration form (Google Form recommended)
- [ ] Add form link to README, website, and outreach materials
- [ ] Set up registration confirmation message/email
- [ ] Share registration link via RoboSUST Club channels, social media, department groups

### Communications
- [ ] Publish announcement on RoboSUST Club pages and social media
- [ ] Distribute poster (see [`outreach/hackathon-poster-text.md`](../outreach/hackathon-poster-text.md))
- [ ] Post on LinkedIn (see [`outreach/linkedin-announcement.md`](../outreach/linkedin-announcement.md))
- [ ] Set up a communication channel for participants (WhatsApp group, Discord, etc.)

### Judging
- [ ] Recruit and confirm 2–3 judges
- [ ] Share [`docs/JUDGING_RUBRIC.md`](./JUDGING_RUBRIC.md) with judges
- [ ] Confirm judges have no conflicts of interest with registered teams
- [ ] Prepare judging score sheets ([`judging/judge-evaluation-sheet.md`](../judging/judge-evaluation-sheet.md))

---

## Phase 2: One Week Before

- [ ] Send reminder to all registered participants with event details
- [ ] Confirm final participant list and team assignments
- [ ] Test all Z.ai APIs and ensure starter kits work correctly
- [ ] Prepare printed/digital schedule for event day
- [ ] Confirm prize money is arranged
- [ ] Prepare certificate templates ([`certificates/templates/`](../certificates/templates/))
- [ ] Brief all volunteers on their roles for the day
- [ ] Run a final check of the venue setup

---

## Phase 3: Day of Event

### Morning Setup
- [ ] Set up venue (tables, power strips, projector, signage)
- [ ] Test internet connection and Z.ai API access
- [ ] Set up registration desk and attendance sheet
- [ ] Brief judges and volunteers

### During the Event
- [ ] Confirm team compositions at registration
- [ ] Distribute any handouts or resources
- [ ] Monitor build sessions and answer participant questions
- [ ] Announce submission deadline at least 30 minutes before it closes
- [ ] Collect all team submissions before the deadline

### Judging & Awards
- [ ] Ensure all judges have scoring sheets
- [ ] Coordinate team demo presentations (2–3 minutes per team)
- [ ] Collect completed judge scoresheets
- [ ] Calculate final scores and determine rankings
- [ ] Prepare tie-break decision if needed (see [`docs/JUDGING_RUBRIC.md`](./JUDGING_RUBRIC.md))
- [ ] Announce winners and distribute prizes
- [ ] Distribute participation certificates (digital or printed)
- [ ] Take group photos for documentation

---

## Phase 4: Post-Event

- [ ] Publish results on this repository and social media
- [ ] Send digital certificates to all participants
- [ ] Update [`participants/team-submissions.md`](../participants/team-submissions.md) with final results
- [ ] Collect feedback from participants and judges
- [ ] Write a post-event summary for LinkedIn/social media
- [ ] Archive this repository for reference and future editions
- [ ] Thank judges, volunteers, Z.ai contacts, and sponsors
